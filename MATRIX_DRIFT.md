# A classic-matrix formalization of the valuation-drift core (`stepA7`)

This note records how one of the four irreducible analytic cores of
`TaoCollatz/Pieces64.idr` — **`stepA7` / `DriftPastTy`**, the 2-adic valuation
drift — is formalized with *classic matrix math*, and exactly what is proved.

The two new modules are

* `TaoCollatz/ValuationDriftMatrix.idr` — the `2x2` mean-drift transfer matrix;
* `TaoCollatz/ValuationVarianceMatrix.idr` — the `3x3` covariance transfer
  matrix (the concentration engine).

Both are `%default total`, have **no** holes and **no**
`believe_me`/`postulate`/`assert_*`/`%foreign`/`idris_crash`/axioms, and the
whole package (`idris2 --build taocollatz.ipkg`, now 70 modules) builds from
scratch.

## What `stepA7` says

```
DriftPastTy =
  (m : Nat) ->
  (good : OddPos -> Bool **
    (AlmostAllOddD good,
     (y : OddPos) -> good y = True ->
       (n : Nat ** (Leq m n, Leq (mult 8 n) (mult 5 (syrValSum n y))))))
```

For each fixed `m`, a density-one set of odd starts reaches the `8/5` drift rate
`8 n <= 5 * S_n(y)` at some time `n >= m`, where `S_n(y)` is the partial sum of
Syracuse 2-adic valuations along the orbit of `y`.  Its own docstring names the
core: *"the typical Syracuse valuation is `2 > 8/5`"* — a law-of-large-numbers /
concentration statement about the valuation random variable.

## The matrix idea

The distribution of a sum of `n` independent valuation increments with law `d`
is the `n`-fold convolution power `convPow d n` (already in
`TaoCollatz.Convolution`).  Its **moment vector** is generated by a fixed,
finite **transfer matrix**, and one convolution step is exactly one matrix step.
This is the classic "the moments of a sum of i.i.d. increments evolve linearly"
picture, made fully explicit and machine-checked.

### The `2x2` mean transfer matrix (`ValuationDriftMatrix`)

The pair `(mass, weightedSum)` (total mass and unnormalised first moment)
transforms under convolution by

```
transferMat d  =  [ mass d          0      ]
                  [ weightedSum d   mass d ]
```

Proved facts:

* `weightedSumConvolve` — the first moment is bilinear:
  `ws(mu * nu) = ws mu * mass nu + mass mu * ws nu` (the second matrix row).
* `momentStep` — `applyMat (transferMat d) (momentVec e) = momentVec (convolve d e)`:
  **one matrix step = one convolution step**.
* `momentVectorMatrix` — `iter n (applyMat (transferMat d)) (1,0) = momentVec (convPow d n)`:
  the moment vector after `n` steps **is** the `n`-th matrix power applied to the
  initial vector `(1,0)`.  This matrix is a scalar times a Jordan block.
* `sumDriftMatrix` — **the `8/5` drift cone is invariant under the transfer
  matrix**: from the per-step drift `8 * mass d <= 5 * weightedSum d`
  (mean valuation `>= 8/5`) it follows, by induction on the transfer-matrix
  recurrence, that

  ```
  8 * n * mass (convPow d n)  <=  5 * weightedSum (convPow d n) .
  ```

  This is exactly the `8/5` drift target of `DriftPastTy` holding **in
  expectation** for the `n`-step valuation sum `S_n`.
* `sumDriftGeoValuation` — the same, specialised to the genuine 2-adic geometric
  valuation measure `geoValuation k` (`k >= 4`), fed by the per-step mean bound
  `ValuationMoment.generalDrift` (`E[a] >= 8/5`, itself derived from the exact
  closed forms `mass = 2^k - 1`, `weightedSum + (k+2) = 2^{k+1}`).

This is the *mean* half of the core, proved end to end with classic matrix math.

### The `3x3` covariance transfer matrix (`ValuationVarianceMatrix`)

To turn the expectation statement into a *density-one* statement one needs
concentration, i.e. control of the variance.  Closing the moment vector with the
raw second moment `weightedSumSq` gives the upper-triangular transfer matrix

```
transferMat3 d  =  [ mass d           0                0      ]
                   [ weightedSum d    mass d           0      ]
                   [ weightedSumSq d  2*weightedSum d  mass d ]
```

Proved facts:

* `expandSq` — the binomial square `(a+b)^2 = a^2 + 2ab + b^2` over `Nat`.
* `weightedSumSqConvolve` — the **covariance-propagation law**
  `m2(mu * nu) = m2 mu * mass nu + 2 * ws mu * ws nu + mass mu * m2 nu`
  (the third matrix row).
* `momentStep3` — one `3x3` matrix step = one convolution step for the full
  `(mass, weightedSum, weightedSumSq)` vector.
* `momentVectorMatrix3` — the full moment vector after `n` steps is the `n`-th
  `3x3` matrix power applied to `(1,0,0)`.

From the Jordan structure of `transferMat3` the second moment of the `n`-step
sum grows like `n * (2nd moment) + n(n-1) * (mean)^2`, i.e. the variance grows
**linearly** in `n` — the `Var(S_n) = O(n)` scaling that is the input to a
Chebyshev concentration bound.

## What remains (the honest gap)

The matrix layer proves the drift-in-expectation and the linear variance
scaling.  Two genuinely analytic steps remain to reach `DriftPastTy` and are
**not** claimed here:

1. **Concentration / normalisation.**  Turning `E[S_n] >= 8n/5` plus
   `Var(S_n) = O(n)` into a density-one statement `S_n(y) >= 8n/5` is a
   Chebyshev argument.  It requires normalising the (unnormalised) `FinDist`
   moments by the total mass `2^k - 1`, which cannot be done inside `Nat`; it
   needs a rational/real measure layer or an equivalent cross-multiplied
   large-deviation estimate beyond the first-moment `TailBound.markov` currently
   available.

2. **Coupling to the genuine dynamics.**  `S_n(y)` is the valuation sum along the
   *actual* Syracuse orbit; the matrix model computes moments of the
   *independent* geometric increment.  Identifying (or dominating) the true
   orbit valuations by the independent model is the equidistribution / coupling
   content of Tao's argument (`DiagonalizationLimit` records why the density
   algebra alone cannot supply it).

Neither gap is papered over: no lemma in these modules assumes them, and the
main-theorem hole `stepA7` in `Pieces64.idr` is untouched.  What the two modules
contribute is the complete, machine-checked *classic-matrix backbone* of the
core — the transfer-matrix mean drift (fully proved, including for the genuine
2-adic measure) and the covariance transfer matrix (the concentration engine) —
on which such an estimate is built.
